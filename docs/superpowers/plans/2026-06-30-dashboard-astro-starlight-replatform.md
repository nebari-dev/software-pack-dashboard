# Dashboard Astro+Starlight Re-platform Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Re-platform the `packs.nebari.dev` dashboard from Hugo to Astro + Starlight, themed by `@nebari/starlight`, hosting the unified Pagefind multisite search, while keeping `generate.py`, `generate_routes.py`, the Cloudflare Worker, and the hourly refresh intact.

**Architecture:** Replace the Hugo site under `site/` with an Astro + Starlight app (Bun) that consumes `@nebari/starlight` from its GitHub release tarball. `generate.py` keeps rendering the landing, now as a Starlight splash `.mdx`. `generate_routes.py` additionally emits the list of pack search-index bundle paths, which `astro.config.mjs` feeds into Starlight's native `pagefind.mergeIndex`. The deploy workflow swaps the Hugo build for an Astro build; the Worker and refresh job are unchanged.

**Tech Stack:** Astro 5 + Starlight 0.33, Bun, `@nebari/starlight` (release tarball), Pagefind (Starlight default + `mergeIndex`), Python 3.12 (`generate.py`/`generate_routes.py`, pytest), Cloudflare Pages + Worker (wrangler-action).

## Global Constraints

- The Astro app uses **Bun** (matches the `@nebari/starlight` ecosystem). No npm/pnpm/yarn lockfiles.
- The site stays a **single splash landing page**; no in-repo docs, no interactive catalog (clean re-platform).
- `@nebari/starlight` is consumed via the **release tarball URL** (see Prerequisite), not npm, until published.
- The Cloudflare Pages deploy directory changes from `site/public` to `site/dist`.
- This repo **auto-deploys to production on push to `main`**; all work lands on the `feat/astro-starlight-replatform` branch and only merges when complete.
- `generate.py`'s `README.md` output is **unchanged**; only the landing render changes.
- No em dashes in any file or commit message. No AI attribution in commits.

- [ ] **Step 3: Implement `build_search_indexes` and write the file**

```python
# generate_routes.py - add this function
def build_search_indexes(routes: dict, extras: dict) -> list[str]:
    """Bundle paths for packs that publish their own Pagefind index.

    A docs-enabled pack has a `<slug> -> https://<slug>.pages.dev` route; we
    exclude __default__ (the dashboard) and the static extra-routes (which are
    not Starlight packs with a Pagefind bundle).
    """
    skip = {"__default__", *extras.keys()}
    return [f"/{slug}/pagefind/" for slug in routes if slug not in skip]
```

In `main()`, after `routes = build_routes(...)`, also write the search-index file:

```python
    search_indexes = build_search_indexes(routes, extras)
    idx_out = root / "site" / "src" / "generated" / "search-indexes.json"
    idx_out.parent.mkdir(parents=True, exist_ok=True)
    idx_out.write_text(json.dumps(search_indexes, indent=2) + "\n")
    print(f"wrote {idx_out} ({len(search_indexes)} search indexes)", file=sys.stderr)
```

**Interfaces:**
- Consumes: the Astro site (Task 1), `generate.py --landing` (Task 2), `generate_routes.py` (Task 3).

- [ ] **Step 1: Replace the Hugo build with Astro/Bun**

Rewrite `.github/workflows/portal-deploy.yml` job steps:

```yaml
name: Portal deploy
on:
  push:
    branches: [main]
  workflow_dispatch:
permissions:
  contents: read
concurrency: { group: portal-deploy, cancel-in-progress: true }
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with: { python-version: '3.12' }
      - run: pip install -r requirements.txt
      - uses: oven-sh/setup-bun@v2

      - name: Render landing content
        run: python generate.py --landing site/src/content/docs/index.mdx
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

      - name: Generate Worker routes + search indexes
        run: python generate_routes.py
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

      - name: Build landing (Astro)
        run: cd site && bun install --frozen-lockfile && bun run build

      - name: Deploy dashboard (landing) to Pages
        uses: cloudflare/wrangler-action@v3
        with:
          apiToken: ${{ secrets.CLOUDFLARE_API_TOKEN }}
          accountId: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}
          command: pages deploy site/dist --project-name=software-pack-dashboard --branch=main

      - name: Deploy routing Worker
        uses: cloudflare/wrangler-action@v3
        with:
          apiToken: ${{ secrets.CLOUDFLARE_API_TOKEN }}
          accountId: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}
          workingDirectory: worker
          command: deploy
```

Key changes from the Hugo version: `peaceiris/actions-hugo` and the `setup-go` step are removed; `oven-sh/setup-bun@v2` added; the landing path is `site/src/content/docs/index.mdx`; `generate_routes.py` runs before the build (so `search-indexes.json` exists); the build is `cd site && bun install --frozen-lockfile && bun run build`; the Pages deploy directory is `site/dist`.
